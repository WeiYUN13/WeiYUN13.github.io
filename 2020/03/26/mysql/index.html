MYSQL数据库学习笔记

0. 数据库的基本常识

RDBMS : relational database management system 关系数据型管理系统

提供API应用程序编程接口

Database：多个数据表构成，每个数据表单由行列分别记录名称和数据域

冗余：存储两倍数据，降低了性能但是提高了数据的安全性

主键：每个数据表只有一个主键，使用主键来查询数据

外键：外键用于关联两个数据表

索引：类似于目录，可以快速访问数据库内的特定信息

复合键：将多列作为一个索引键

DISTINCT消除重复的属性值

多条件排序

HAVING 条件表达式用于限制分组结果的显示

WITH ROLLUP 对查询记录做总和：数字相加，字符串集合

LIMITE限制查询记录的数量：LIMITE 初始位置，记录数

集合函数：COUNT(),SUM(),AVG(),MAX(),MIN()

3.SQL语句的分类

3.1 DDL(data definition language)数据定义语言：创建，删除，修改结构

数据库操作：

show databases/tables; 查看所有数据库/某数据库下所有的数据表

use 数据库名；切换数据库

create database 数据库名； 创建数据库

drop databases 数据库名； 删除数据库

数据表操作：

create table 数据表名 （属性名1 数据类型 约束条件，

属性名2 数据类型 约束条件，

…….

属性名n 数据类型 约束条件

）

查看表结构：

desc 数据表名；

查看创建表的sql语句：

show create table 数据表名;

删除表：

drop table 表名；

修改字段类型的修饰：

alter table 表名 modify 字段名 修改内容;

修改字段名：

alter table 表名 change 旧字段名 新字段名 修饰内容;

增加字段：

alter table 表名 add 属性名1 数据类型 [约束条件] [FIRST |AFTER 属性2]

定义新增字段位于表的位置，由first定义到表第一个字段，AFTER 属性定义到某属性后

默认定义到表的最后一个

删除表的某一列：

alter table 表名 drop 字段名

修改表名：

alter table 旧表名 rename (to) 新表名

更改表的引擎：alter table 表名 engine=存储引擎名；

删除外键：alter table 表名 drop foreign key 外键名

删除某个父表前必须先把依赖于该父表的外键约束删除，最后再删除父表drop table 父表名

3.2 DML(data manipulation language)数据操作语言：增，删，改表的记录

查询所有数据：select * from 表名

插入数据：insert into 表名 （字段1，字段2…..） values (字段值1，字段值2……)

更新数据：update 表名 set 字段名1=值1，字段名2=值2 where 条件

删除数据：delete from 表名 where 条件

truncate 表名（重置表）

3.3 DCL(data control language)数据控制语言：用户的创建与授权

设置root的ip访问，密码的设置与重置，普通用户权限设置与删除

3.4 DQL(data query language)数据查询语言：查询数据

查询数据：select 字段名1，字段名2…… from 表名 [where 条件]

过滤重复的列值：select distinct 列名1 from 表名

连接concat：

select concat (列名1，列名2) as 新名字 from 表名

select concat_ws (‘分隔符’, 列名1，列名2) as 新名字 from 表名

起别名as：select 列名1 as 别名 from 表名

模糊查询LIKE：select 列名 from 表名 where 列名 LIKE ‘%字符串%’

%字符串 ：左模糊，字符串%：右模糊，%字符串%：全模糊

排列：select 字段1，字段2….. from 表名 order by 字段名 ASC（升序，默认，DESC 降序）

聚合函数：COUNT,SUM,AVG,MAX,MIN

select sum(列名) from 表名 查询此列的和

分组查询：select * from 表名 GROUP BY 列名 HAVING count(*)>3

连接查询

内连接查询inner join：从多个表中连接查询

select s.name,m.mark,age from student as s inner join mark as m where m.stu_id = s.id;

左连接查询left join: 先列左边的表为基准

select s.name,m.mark,age from student as s left join mark as m on m.stu_id = s.id;

右连接查询right join:先列右边的表为基准

select s.name,m.mark,age from student as s right join mark as m on m.stu_id = s.id;

联合查询 列数一致

select 字段 from 表名 union all select 字段1 from 表名1

子查询: 查询中嵌套查询

select id from student where id in (select stu_id from mark);

限制查询limit：限制查询条数，查询中间的条数

select * from 表名 limit 条数1，条数2

4.3 MYISAM与InnoDB的区别

MYISAM 支持全文索引（full text），不支持事务，表级锁（单次操作），崩溃恢复支持不好

InnoDB 支持事务，不支持全文索引，但5.6后开始支持，行级锁（行之间独立，性能快），所以推荐InnoDB

5.1 位（比特），字节，字符

6. DTL事务控制语言

事务：一个SQL语句就是一个事务（仅InnoDB支持事务）

事务可以确保 一组SQL语句要么都成功要么都失败（失败下回滚）

start transaction; 开始事务

多个SQL语句

若失败，使用 rollback语句。

若成功，使用 commit语句

事务的特性：原子性（全做或全不做），数据一致性（逻辑合理），隔离性（每个事务独自执行），持久性（磁盘数据更改而不是内存中更改）

6.2 视图创建:视图不可以修改！只能查看

视图是由一个或者多个表中导出来的，虚拟存在的表。视图中的数据依赖原先的表中的数据。

视图有利于增强用户只关心自己有用的数据，原表数据的变化会导致依赖它的视图的变化。

CASCADED 保证新视图满足父视图的约束条件

AS SELECT 语句，用户权限可以select的权限

查询用户的select权限和create view权限：

select select_priv,create_view_priv from mysql.user where user=’root’;

单表上创建视图： create view 视图名

视图的属性是select查询的所有的表中所有的属性

从单表的某些属性中创建视图：

create view

8. 索引

索引 组合数据库表的一列或者多列信息，通过索引可以加快数据库的查询速度

储存引擎 定义表中的最大索引数和字节数，，以及存储类型,B型树或者哈希索引

添加索引：alter table 表名 add index/unique/fulltext [索引名] 字段名

删除索引：alter table 表名 drop index 索引名

主键索引 primary key：每个表只有一个主键，primary key (字段名1，字段名2…..)，主键值不可以重复

唯一索引 unique : unique（字段名），限制值不可以重复

普通索引 index,key : key(字段名)

查看表的索引：show index from 表名\G

全文索引fulltext：加速模糊查询速度

select * from 表名 where match (全文索引字段) against (‘查询词’)；

外键约束：外键字段属性必须与主键一致 （确保多个表的一致性 ）

foreign key(子表字段) references 父表(主键)

设置表的属性默认值：属性 数据类型 DEFAULT 默认值

标识属性为表的主键

主键的要求：主键是唯一的，主键字段的值也是唯一的，能够快速查找到表中的信息

外键的概念：

外键必须依赖于已存在的父表的主键，

外键建立了与父表的关联关系

CONSTRAINT 外键别名 FOREIGN KEY(属性1，属性2…………)

REFERENCES 表名 (属性1，属性2………..)

外键将子表的属性对应到对应父表的主键上，对应属性的数据类型必须一致

外键可以防止破坏表连接的动作，以及约束子表的非法插入数据

2. union 和 union all

UNION 操作符用于合并两个或多个 SELECT 语句的结果集。

请注意，UNION 内部的 SELECT 语句必须拥有相同数量的列。列也必须拥有相似的数据类型。同时，每条 SELECT 语句中的列的顺序必须相同。

UNION ALL 命令和 UNION 命令几乎是等效的，不过 UNION ALL 命令会列出所有的值。

3. CHECK约束

CHECK 约束用于限制列中的值的范围。

如果对单个列定义 CHECK 约束，那么该列只允许特定的值。

Id_P int NOT NULL CHECK (Id_P>0) 限制 Id_P的值大于0
