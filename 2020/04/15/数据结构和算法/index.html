数据结构, 算法

数据结构分为逻辑结构和物理结构

逻辑结构：

  • 集合结构：数据之间同属一个集合，无相关的关系
  • 线性结构：元素之间是一对一的关系
  • 树形结构：一对多的层次关系
  • 图形结构：元素之间是多对多的关系（如关系网）

物理结构：如何把数据存储到计算机的存储器（内存）中

  • 顺序存储结构：数据存放在地址连续的内存单元中
  • 链式存储结构：数据元素可以存储在任意的存储单元内

1. 算法的复杂度

  • 算法的复杂度分为时间复杂度和空间复杂度
  • 函数的渐进增长：判断函数的增长速度（最高项的阶数）
  • 算法的最好情况，最坏情况与平均情况
  • 算法的时间复杂度与空间复杂度是可以互换的

2. 线性表（List）

定义元素的前驱元素和后续元素（除第一个元素和最后一个外，有且只有一个）
抽象数据类型
指的是编程者在设计软件时自己定义的数据类型（比如定义某个类）
抽象数据类型实例：实现数据类型的相关属性和方法

1） 线性表的顺序存储结构

使用地址连续的存储单元存储线性表的数据元素
顺序存储结构具有随机存储结构的特点，存储是指将元素存入线性表的某个位置

随机存储结构：存储的时间性能为O(1)

线性表的插入（删除）操作：将位置i后续元素以此向后移动一个位置，复杂度为O(n)

顺序存储结构的优缺点：

  • 优点：
    元素间的逻辑关系不需要额外的存储空间
    快速存取任意位置的元素
  • 缺点：
    插入和删除操作的复杂度为O(n)
    磁盘空间碎片化

2）线性表的链式存储结构

顺序存储结构的插入和存储时间复杂度为O(n)，如何解决这一问题呢？
链式存储结构使用任意位置的存储单元，每个存储结构中需要存储后继元素的存储地址（指针）
结点分为数据域和指针域，只有一个指针域时称为单链表

头结点和头指针的区别

头指针指向链表的第一个存储位置
头结点的数据域无意义

单链表的读取时间复杂度为O(n)，插入和删除的复杂度为O(1)，因此需要频繁插入或者删除数据时，单链表具有明显的优势

单链表和顺序存储的对比
存储分配方式，时间性能， 空间性能

3）静态链表

用数组描述的链表叫做静态链表，描述方法叫做游标实现法

静态链表的第一个和最后一个元素的data不存放数据，第一个元素指向备用链表第一个节点的下标。最后一个元素指向存放第一个有数值的元素的下标，相当于头结点的作用。

静态链表实现单链表的操作
插入：将B插入到A后，即将B存储至备用链表的第一个元素，将A的游标改为B的下标，将B的游标改为原先A的游标（即原先A指向的下标）。
删除：待删除元素B的游标改为B元素指向的下标的游标（B.next = B.next.next），将元素B加入备用链表中，将下标0的游标指向B的下标，B的游标指向原先0下标对应的游标。

静态链表的优缺点
优点：

  • 插入和删除时只需要修改游标，不需要移动元素

缺点：

  • 失去了顺序存储的随机存取的特性
  • 没有解决连续分配数组带来的表长难以确定的问题

例题：寻找链表的中间节点

4）循环链表

循环链表：单链表的尾节点指向头节点（rear.next = head）

循环链表常用尾指针，而不是头指针

例题：判断单链表中是否有环

3. 栈和队列

1) 栈Stack

后进先出(last in first out)的数据结构，只在表尾进行删除和插入操作，将表尾称为栈的栈顶（TOP），将表头称为栈的栈底（BOTTOM）

  • 栈的插入称为push，称为进栈和压栈
  • 栈的删除称为pop，称为出栈或者弹栈
  • 栈分为顺序存储结构和栈的链式存储结构
    链式存储结构中栈顶为单链表的头指针

逆波兰表达式:
(a+b)c的逆波兰式为ab+c，假设计算机把ab+c*按从左到右的顺序压入栈中，并且按照遇到运算符就把栈顶两个元素出栈，执行运算，得到的结果再入栈的原则来进行处理

中缀表达式转后缀表达式：运算数位置不变，改变的是运算符的位置
需要一个存放操作符的栈op_stack，输出结果的列表output
步骤：
从左到右遍历表达式：

  1. 若是数字，直接加入到output
  2. 若是操作符，比较该操作符和stack中操作符的优先级，若优先级大于stack中的，则压入到stack中
    否则，将stack中优先级大于或等于该操作符优先级的所有操作符加入到output中，然后压入stack中
  3. 若是左括号，压入到stack中
  4. 若是右括号，将stack中所有左括号前面的操作符加入到output中
    重复上面的步骤

2) 队列（queue）

先进先出(first in first out)的数据结构，类似于排队的机制。队列是一种只允许在一段进行插入在另一端进行删除的线性表。队列通常使用链表来实现，称为链队列。使用链表实现的队列，插入和删除的操作时间复杂度均为O(1)。而实现顺序存储结构时，出队列的时间复杂度是O(n)。

解决假溢出的方法：循环队列

4. 递归与分治算法

1）递归

递归是一种效率较低的算法，建议使用循环替换递归

递归过程是一种调用自身的函数，在调用的过程中存在转入子程序的过程。在每次转子前需要保护现场，则将相应参数和中间结果压入系统堆栈，而在子程序返回的时候，需要恢复现场，则将之前压栈的数据从系统堆栈弹出，因此，递归过程存在隐含的堆栈操作，而且子程序的调用过程满足堆栈先进后出的特性。递归函数调用的太深，需要太多的内存，递归里用到的局部变量存储在堆栈中，堆栈的访问效率高，速度快，但空间有限，递归太多变量需要一直入栈而不出栈，导致需要的内存空间大于堆栈的空间。

例题：斐波那契数列 https://leetcode-cn.com/problems/fibonacci-number/

2） 分治思想

当我们求解某些问题时，由于这些问题要处理的数据相当多，或求解过程相当复杂，使得直接求解法在时间上相当长，或者根本无法直接求出。对于这类问题，我们往往先把它分解成几个子问题，找到求出这几个子问题的解法后，再找到合适的方法，把它们组合成求整个问题的解法。如果这些子问题还较大，难以解决，可以再把它们分成几个更小的子问题，以此类推，直至可以直接求出解为止。这就是分治策略的基本思想

例题：汉诺塔问题 https://leetcode-cn.com/problems/hanota-lcci/solution/tu-jie-yi-nuo-ta-de-gu-shi-ju-shuo-dang-64ge-pan-z/
注意，当你在思考这个问题的时候，就将最初的 n 个盘子从 A 移到 C 的问题，转化成了将 n - 1 个盘子从 A 移到 C 的问题， 依次类推，直至转化成 1 个盘子的问题时，问题也就解决了。这就是分治的思想。

而实现分治思想的常用方法就是递归。不难发现，如果原问题可以分解成若干个与原问题结构相同但规模较小的子问题时，往往可以用递归的方法解决。具体解决办法如下：
n = 1 时，直接把盘子从 A 移到 C；
n > 1 时，
先把上面 n - 1 个盘子从 A 移到 B（子问题，递归）；
再将最大的盘子从 A 移到 C；
再将 B 上 n - 1 个盘子从 B 移到 C（子问题，递归）。

八皇后问题 https://leetcode-cn.com/problems/eight-queens-lcci/

5）字符串

字符串的存储结构：分为顺序存储和链式存储结构

例题： https://leetcode-cn.com/problems/implement-strstr/

  • BF算法（Brute Force）
    朴素的模式匹配算法，BF算法的思想：将目标串S的第一个字符与模式串T的第一个字符进行匹配，若相等，则继续比较S的第二个字符和 T的第二个字符；若不相等，则比较S的第二个字符和T的第一个字符，依次比较下去，直到得出最后的匹配结果。BF算法是一种蛮力算法。子串的定位操作称为串的模式匹配。
  • KMP算法（克努特-莫里斯-普拉斯算法）https://leetcode-cn.com/problems/implement-strstr/solution/python-bao-li-fa-kmp-by-xxinjiee/
    KMP算法是一种改进的字符串匹配算法，由D.E.Knuth，J.H.Morris和V.R.Pratt提出的，因此人们称它为克努特—莫里斯—普拉特操作（简称KMP算法）。KMP算法的核心是利用匹配失败后的信息，尽量减少模式串与主串的匹配次数以达到快速匹配的目的。具体实现就是通过一个next()函数实现，函数本身包含了模式串的局部匹配信息。KMP算法的时间复杂度O(m+n)。KMP算法避免了不必要的回溯。问题是由模式串决定的，而不是目标串。

next数组用于在模式匹配串失效的时候，指导应该使用模式串的哪个元素进行下一轮的匹配。
next部分匹配表为对应元素前后缀共同元素的个数，以”ABCDABD”为例。

“A”的前缀和后缀都为空集，共有元素的长度为0；
“AB”的前缀为[A]，后缀为[B]，共有元素的长度为0；
“ABC”的前缀为[A, AB]，后缀为[BC, C]，共有元素的长度0；
“ABCD”的前缀为[A, AB, ABC]，后缀为[BCD, CD, D]，共有元素的长度为0；
“ABCDA”的前缀为[A, AB, ABC, ABCD]，后缀为[BCDA, CDA, DA, A]，共有元素为”A”，长度为1；
“ABCDAB”的前缀为[A, AB, ABC, ABCD, ABCDA]，后缀为[BCDAB, CDAB, DAB, AB, B]，共有元素为”AB”，长度为2；
“ABCDABD”的前缀为[A, AB, ABC, ABCD, ABCDA, ABCDAB]，后缀为[BCDABD, CDABD, DABD, ABD, BD, D]，共有元素的长度为0。
